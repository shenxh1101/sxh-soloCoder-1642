## 1. 产品概述

律所客户案件与文书管理系统，面向中小型律师事务所，提供客户信息管理、案件档案管理、案件进度跟踪、文书模板生成与编辑、案件日程提醒、数据统计与导出等一站式功能，解决律所案件管理混乱、文书重复编写、日程遗漏等痛点。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 律师 | 系统管理员分配账号 | 客户录入、案件管理、文书编辑、日程管理、数据统计 |
| 管理员 | 系统初始账号 | 全部权限 + 律师账号管理 + 系统配置 |

### 2.2 功能模块

1. **工作台**：待办日程、近期案件、快捷入口、数据概览
2. **客户管理**：客户列表、客户详情、客户新增/编辑
3. **案件管理**：案件列表、案件详情、案件新增/编辑、进度跟踪
4. **文书管理**：文书模板列表、文书生成、在线编辑、下载Word
5. **日程管理**：日程日历视图、日程提醒、日程新增/编辑
6. **数据统计**：案件类型统计、状态分布、导出报表

## 3. 核心流程

### 3.1 案件管理主流程

律师录入客户信息 → 为客户创建案件档案 → 填写案由、对方当事人等信息 → 添加案件进度阶段 → 为案件安排日程 → 选择文书模板生成文书 → 编辑并下载文书 → 导出统计报表

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深藏青 (#1B2A4A) 搭配暖金 (#C9A96E)，体现律所的专业与庄重
- **辅助色**：象牙白 (#F5F0EB) 作为背景，浅灰 (#E8E3DE) 作为卡片背景
- **按钮风格**：圆角4px，主按钮深藏青底色白字，次要按钮描边样式
- **字体**：标题使用 Noto Serif SC 衬线体，正文使用 Noto Sans SC 无衬线体
- **布局风格**：左侧固定导航栏 + 右侧内容区，卡片式布局
- **图标风格**：线性图标，统一2px描边
- **整体氛围**：专业、沉稳、高效，兼具现代感与传统法治文化底蕴

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 工作台 | 数据概览 | 4个统计卡片，深藏青背景配金色数字，微妙投影 |
| 工作台 | 待办日程 | 白色卡片列表，左侧色条标识紧急程度，日期标签 |
| 工作台 | 近期案件 | 紧凑表格行，状态徽标，右侧箭头跳转 |
| 客户列表 | 客户表格 | 斑马纹表格，操作列按钮组，顶部搜索栏 |
| 客户详情 | 基本信息 | 双栏信息展示，标签式案件类型，编辑入口 |
| 案件列表 | 案件表格 | 多条件筛选栏，状态筛选标签页，表格分页 |
| 案件详情 | 进度跟踪 | 竖向时间线，节点圆点+连接线，阶段卡片展开详情 |
| 案件详情 | 关联文书 | 文书卡片网格，模板图标+文书名+创建日期 |
| 文书生成 | 在线编辑 | 左侧模板变量面板 + 右侧富文本编辑器 |
| 日程管理 | 日历视图 | 月历格子，有日程的日期显示色点，点击展开详情 |
| 数据统计 | 图表区域 | 柱状图+饼图，图表交互高亮，右上角导出按钮 |

### 4.3 响应式设计

- 桌面优先设计，最小宽度1200px完整体验
- 平板端（768px-1200px）侧边栏收起为图标模式
- 移动端（<768px）侧边栏隐藏，顶部汉堡菜单，表格改为卡片列表
